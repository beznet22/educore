# CMS Domain — Entities

Entities have identity and lifecycle but are not aggregate roots. They
are loaded and persisted only through their aggregate root.

## NewsImage

**Identity:** `NewsImageId(SchoolId, Uuid)`
**Owner:** `News`

The `image` and `image_thumb` references for a `News` entry. Both are
`FileReference`s; the thumb is generated by an image processing port.

## PageSetting

**Identity:** Embedded in `Page`
**Owner:** `Page`

A typed JSON value object holding per-page settings. Schema is
versioned and consumer-defined; the domain validates that the JSON is
well-formed.

## ContentShareListAudience

**Identity:** `ContentShareListAudienceId(SchoolId, Uuid)`
**Owner:** `ContentShareList`

A materialized audience descriptor: a `send_type` (`G`, `C`, `I`, `P`)
plus the matching `content_ids`, `gr_role_ids`, `ind_user_ids`, and
`section_ids` lists. Captured at dispatch time.

## ContentShareListContent

**Identity:** `ContentShareListContentId(SchoolId, Uuid)`
**Owner:** `ContentShareList`

A join between a `ContentShareList` and a `Content`. The list has
many contents; each content appears in many lists.

## ContentAvailability

**Identity:** Embedded in `Content`
**Owner:** `Content`

A typed struct holding `available_for_role`, `available_for_class`,
and `available_for_section` references.

## TeacherUploadContentScope

**Identity:** Embedded in `TeacherUploadContent`
**Owner:** `TeacherUploadContent`

A typed struct holding the class-section, course, parent course,
chapter, and lesson scope of a teacher-uploaded content.

## UploadContentScope

**Identity:** Embedded in `UploadContent`
**Owner:** `UploadContent`

A typed struct holding the role, class, and section availability.

## PageRevision

**Identity:** `PageRevisionId(SchoolId, Uuid)`
**Owner:** `Page`

A historical snapshot of a `Page` body. Revisions are append-only and
allow reverting the page to a prior state.

## NewsRevision

**Identity:** `NewsRevisionId(SchoolId, Uuid)`
**Owner:** `News`

A historical snapshot of a `News` body. Revisions are append-only.

## ContentRevision

**Identity:** `ContentRevisionId(SchoolId, Uuid)`
**Owner:** `Content`

A historical snapshot of a `Content` body. Revisions are append-only.

## CoursePageRelation

**Identity:** `CoursePageRelationId(SchoolId, Uuid)`
**Owner:** `CoursePage`

A typed edge between two `CoursePage` aggregates: `parent_id` →
`CoursePageId`. Models the course hierarchy.

## HomeSliderOrder

**Identity:** Embedded in `HomeSlider`
**Owner:** `HomeSlider`

A display order for the home-page slider. The order is held on the
aggregate as an integer; a port adapter renders the order.

## TestimonialImage

**Identity:** Embedded in `Testimonial`
**Owner:** `Testimonial`

The image `FileReference` for a testimonial. Held on the aggregate
as a value; not a separate entity.
